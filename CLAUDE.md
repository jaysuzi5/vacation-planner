## Project Overview

**Vacation Planner** — Private per-user vacation planning and expense tracking app. Users log in with Google, manage their own trips (status: review/booked/taken), enter budget vs. actual per category, build daily itineraries, and log line-item expenses per day.

**Status:** Active development

**Stack:**
- Python 3.12 / Django 6.0
- PostgreSQL (shared k8s instance) — DB: `vacation_planner`
- django-allauth (email + Google OAuth)
- Bootstrap 5 + custom CSS (clean card-based design, modeled on unscripted-living)
- WhiteNoise for static files
- OpenTelemetry for observability
- Docker + Kubernetes (namespace: `vacation-planner`)
- Package manager: uv

## Development Setup

```bash
# Install dependencies
uv sync

# Set up environment
cp .env.example .env  # fill in real values

# Run migrations
uv run python manage.py migrate

# Create superuser
uv run python manage.py createsuperuser

# Run dev server
uv run python manage.py runserver

# Add a package
uv add <package-name>

# Check for issues
uv run python manage.py check
```

**Environment variables:** Copy `.env.example` to `.env`. Database host/port for local dev uses the k8s external address (192.168.86.201:30004).

## Architecture

Single Django app (`planner/`) with three models:

- `Vacation` — user-scoped trip with location, dates, status, budget/actual per category, trip notes
- `Day` — daily itinerary entry (date, lodging, excursion, notes) linked to Vacation
- `Expense` — line-item spending (description, category, amount) linked to Day

**User isolation:** All views inherit from `LoginRequiredMixin` and enforce `request.user == vacation.user`. No user can see another's data. This is private, not public.

### Key entry points
- `config/urls.py` — root URL config (admin + allauth + planner)
- `planner/views.py` — all application views
- `planner/models.py` — Vacation, Day, Expense models
- `config/settings.py` — all configuration (reads from `.env`)
- `config/middleware.py` — JSON request/response logging

### Non-obvious conventions
- All credentials from `.env` (local) or k8s SealedSecrets (production). Never hardcode.
- `k8s/temp.yaml` is gitignored plain-text secrets — delete after running kubeseal.
- Cloudflare terminates TLS — `SECURE_SSL_REDIRECT = False` in production is correct and intentional.

### Files to never edit directly
- `staticfiles/` — generated by `collectstatic`
- `planner/migrations/` — generated by `makemigrations`, don't hand-edit

## Deployment

```bash
# Build and push multi-arch Docker image
docker buildx build --platform linux/amd64,linux/arm64 -t jaysuzi5/vacation-planner:latest --push .

# Apply k8s manifests
kubectl apply -f k8s/deployment.yaml

# Restart deployment
kubectl rollout restart deployment vacation-planner -n vacation-planner

# Check pods
kubectl get all -n vacation-planner

# Exec into pod
kubectl exec -n vacation-planner -it <pod-name> -- /bin/sh

# Run management command in pod
kubectl exec -n vacation-planner -it <pod-name> -- uv run python manage.py <command>
```

**Domain:** vacations.jaycurtis.org (Cloudflare tunnel)

**Secrets:** Managed as SealedSecrets in `k8s/secrets.yaml`.
